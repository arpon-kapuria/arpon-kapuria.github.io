Motivation

Most RAG pipelines fail silently.

Retrieved chunks appear relevant, the answer sounds convincing, yet factual consistency collapses under multi-hop reasoning.

KEY IDEAS
  • Retrieval should be iterative
  • Verification before generation
  • Embeddings alone are insufficient
  • Recursive refinement improves grounding

Architecture

The pipeline introduces a recursive verification module before final response generation.

Recursive Verification

verification.py 

def recursive_verify(query, retrieved_docs):

    verified_docs = []

    for doc in retrieved_docs:

        score = consistency_check(query, doc)

        if score > 0.8:
            verified_docs.append(doc)

    return verified_docs
